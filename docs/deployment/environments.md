# Environment Configuration

This document explains how to configure and run the WorshipHub UI app in different environments.

## Available Environments

- **Development** (default): Local development with emulator/localhost
- **Staging**: Testing environment with staging backend
- **Production**: Production environment with live backend

## Configuration Files

- `lib/core/config/environment.dart`: Environment enum and configuration
- `lib/core/config/app_config.dart`: Environment-specific URLs and settings
- `lib/main.dart`: App initialization with environment setup

## Running in Different Environments

### Development (Default)

```bash
flutter run
```

This uses:
- Android Emulator: `http://10.0.2.2:9090`
- Web: `http://localhost:9090`
- WebSocket: `ws://10.0.2.2:9090/ws/chat` (Android) or `ws://localhost:9090/ws/chat` (Web)

### Staging

```bash
flutter run --dart-define=ENV=staging
```

This uses:
- API: `https://staging-api.worshiphub.com`
- WebSocket: `wss://staging-api.worshiphub.com/ws/chat`

**Note**: Update the staging URLs in `app_config.dart` when staging environment is available.

### Production

```bash
flutter run --dart-define=ENV=production --release
```

This uses:
- API: `https://api.worshiphub.com`
- WebSocket: `wss://api.worshiphub.com/ws/chat`

**Note**: Update the production URLs in `app_config.dart` when production environment is available.

## Environment-Specific Settings

### Log Levels

- **Development**: `debug` - All logs including debug messages
- **Staging**: `info` - Info, warning, and error logs
- **Production**: `warning` - Only warning and error logs

### Security

- **Development**: Uses HTTP for local development
- **Staging/Production**: Uses HTTPS and WSS (secure WebSocket)

## Firebase Configuration

Firebase configuration is managed separately through `firebase_options.dart` which is generated by FlutterFire CLI.

To configure Firebase for different environments:

```bash
# Development
flutterfire configure --project=worshiphub-dev

# Staging
flutterfire configure --project=worshiphub-staging

# Production
flutterfire configure --project=worshiphub-prod
```

## Environment Variables (Future Enhancement)

For sensitive configuration (API keys, secrets), consider using:

1. `.env` files with `flutter_dotenv` package
2. Build-time variables with `--dart-define`
3. Platform-specific configuration files

Example:

```bash
flutter run --dart-define=API_KEY=your_api_key --dart-define=ENV=production
```

## Troubleshooting

### Wrong Environment

If the app is using the wrong environment:

1. Check the `main.dart` initialization
2. Verify `--dart-define=ENV=<environment>` is passed correctly
3. Clean and rebuild: `flutter clean && flutter pub get && flutter run`

### Connection Issues

- **Development**: Ensure backend is running on `localhost:9090`
- **Staging/Production**: Verify URLs in `app_config.dart` are correct
- **Android Emulator**: Use `10.0.2.2` instead of `localhost`
- **iOS Simulator**: Use `localhost` directly

## Security Best Practices

1. **Never commit sensitive data** (API keys, tokens) to version control
2. **Use environment variables** for sensitive configuration
3. **Rotate credentials** regularly
4. **Use HTTPS/WSS** in staging and production
5. **Implement certificate pinning** for production (future enhancement)
