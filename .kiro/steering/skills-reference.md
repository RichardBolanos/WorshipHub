# Skills Disponibles

Este proyecto tiene skills especializados que deben ser considerados automáticamente según el contexto de trabajo.

## Skills Activos

Los siguientes skills están disponibles y deben ser activados cuando el trabajo esté relacionado con sus áreas:

### Backend y API
- **backend-api-patterns**: Patrones de implementación de backend y APIs para escalabilidad, seguridad y mantenibilidad. Usar cuando se construyan APIs, servicios e infraestructura backend.
- **backend-testing**: Estrategias y patrones de testing para aplicaciones backend.

### Frontend y Mobile
- **flutter**: Desarrollo cross-platform con Flutter y Dart para iOS, Android, Web, Desktop.
- **flutter-development**: Desarrollo de aplicaciones Flutter con mejores prácticas y patrones modernos.

### Tecnologías Específicas
- **kotlin-springboot**: Desarrollo de aplicaciones backend con Kotlin y Spring Boot.

### Procesos y Metodología
- **git-workflow**: Flujos de trabajo con Git, branching strategies y mejores prácticas.
- **testing-strategies**: Estrategias generales de testing y QA.
- **find-skills**: Ayuda a descubrir e instalar skills del ecosistema de agent skills. Usar cuando el usuario busque funcionalidades específicas o pregunte "¿hay un skill para X?"

## Activación Automática

Los skills deben activarse automáticamente cuando:
- Se trabaje en archivos relacionados con su dominio
- Se mencionen tecnologías o patrones cubiertos por el skill
- Se solicite implementar funcionalidades dentro de su alcance

## Ubicación de Skills

Todos los skills están en: `.kiro/skills/[nombre-skill]/SKILL.md`

Para ver el contenido completo de un skill específico, consulta su archivo SKILL.md correspondiente.

## Nota Importante

Esta es la única ubicación de skills en el proyecto. Se han eliminado las carpetas duplicadas de otros IDEs (.agent, .agents, .windsurf, .junie, skills/) para mantener una estructura limpia y enfocada en Kiro.
